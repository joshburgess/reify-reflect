# Guide 4: The `#[reifiable]` Proc Macro

## The Problem

Guide 3 showed how `NatCallback` and `reify_nat` dispatch a runtime value
into a const-generic context. But for each new trait you want to dispatch,
you'd need to manually write the 256-arm match table.

The `#[reifiable]` proc macro automates this. Annotate a trait, and it
generates the dispatch functions for you.

## Basic Usage

```rust
use const_reify_derive::reifiable;

#[reifiable(range = 0..=255)]
trait ModArith {
    fn pow_mod<const N: u64>(&self, base: u64, exp: u64) -> u64;
    fn name(&self) -> &str;  // not const-generic — left alone
}
```

The macro:
1. **Leaves the trait unchanged** — it's still a normal trait
2. **Generates `reify_pow_mod`** — a dispatch function for the const-generic method
3. **Skips `name`** — it has no const generic, so no dispatch needed

## What Gets Generated

For each method with a `<const N: u64>` parameter, the macro generates
a standalone function:

```rust
// You write:
#[reifiable(range = 0..=255)]
trait ModArith {
    fn pow_mod<const N: u64>(&self, base: u64, exp: u64) -> u64;
}

// The macro generates (conceptually):
fn reify_pow_mod<T: ModArith>(val: u64, obj: &T, base: u64, exp: u64) -> u64 {
    match val {
        0 => obj.pow_mod::<0>(base, exp),
        1 => obj.pow_mod::<1>(base, exp),
        // ... 254 more arms ...
        255 => obj.pow_mod::<255>(base, exp),
        _ => panic!("out of range 0..=255"),
    }
}
```

The naming convention is `reify_<method_name>`.

## Implementing the Trait

You implement the trait normally — the `#[reifiable]` attribute only
affects the trait definition, not its impls:

```rust
struct FastModArith;

impl ModArith for FastModArith {
    fn pow_mod<const N: u64>(&self, base: u64, exp: u64) -> u64 {
        if N == 0 { return 0; }
        // N is a const generic — the compiler can optimize per-modulus
        let mut result = 1u64;
        let mut b = base % N;
        let mut e = exp;
        while e > 0 {
            if e % 2 == 1 { result = result * b % N; }
            b = b * b % N;
            e /= 2;
        }
        result
    }

    fn name(&self) -> &str { "fast" }
}
```

## Using the Generated Dispatch

```rust
let modulus: u64 = get_modulus_from_user();  // runtime value!

// reify_pow_mod dispatches modulus into the const generic N
let result = reify_pow_mod(modulus, &FastModArith, 3, 100);
println!("3^100 mod {} = {}", modulus, result);
```

The runtime `modulus` becomes the compile-time `N` inside `pow_mod`.
Each of the 256 monomorphizations is a fully specialized modular
exponentiation.

## A Complete Example

```rust
use const_reify_derive::reifiable;

#[reifiable(range = 0..=255)]
trait Checker {
    fn is_prime_witness<const N: u64>(&self, a: u64) -> bool;
    fn description(&self) -> &str;
}

struct MillerRabin;

impl Checker for MillerRabin {
    fn is_prime_witness<const N: u64>(&self, a: u64) -> bool {
        if N < 2 { return false; }
        if N == 2 || N == 3 { return true; }
        if N % 2 == 0 { return false; }

        // Factor N-1 as 2^r * d
        let mut d = N - 1;
        let mut r = 0u32;
        while d % 2 == 0 { d /= 2; r += 1; }

        // Compute a^d mod N
        let mut x = {
            let mut result = 1u64;
            let mut base = a % N;
            let mut exp = d;
            while exp > 0 {
                if exp % 2 == 1 { result = result * base % N; }
                base = base * base % N;
                exp /= 2;
            }
            result
        };

        if x == 1 || x == N - 1 { return true; }
        for _ in 0..r - 1 {
            x = x * x % N;
            if x == N - 1 { return true; }
        }
        false
    }

    fn description(&self) -> &str { "Miller-Rabin" }
}

fn main() {
    let checker = MillerRabin;

    // Test primality for runtime-determined values
    for n in [2, 3, 7, 11, 13, 15, 17, 100, 127, 131, 251] {
        // reify_is_prime_witness was auto-generated by #[reifiable]
        let is_prime = reify_is_prime_witness(n, &checker, 2);
        println!("{}: {}", n, if is_prime { "probably prime" } else { "composite" });
    }
}
```

## Configurable Range

The range doesn't have to be 0..=255:

```rust
#[reifiable(range = 0..=15)]  // only 16 monomorphizations
trait SmallOp {
    fn compute<const N: u64>(&self) -> u64;
}
```

Smaller ranges compile faster. The macro enforces a maximum of 1024
to prevent accidental compile-time explosions.

## What the Macro Handles

| Feature | Supported? |
|---|---|
| `&self` methods | Yes |
| `&mut self` methods | Yes |
| Extra parameters (`x: f64`, etc.) | Yes |
| Lifetime parameters on methods | Yes |
| Type parameters on methods | Yes |
| Generic traits (`trait Foo<T>`) | Yes |
| Default method impls | Yes (transparent) |
| Multiple const-generic methods | Yes (one dispatch fn each) |
| Non-const-generic methods | Skipped (no dispatch generated) |
| Return types depending on N | Error (V1 limitation) |
| Multiple const params per method | Error (V1 limitation) |
| Async methods | Not yet (V2) |

## The `&mut self` Case

```rust
#[reifiable(range = 0..=31)]
trait Accumulator {
    fn add<const N: u64>(&mut self);
}

struct Counter { total: u64 }

impl Accumulator for Counter {
    fn add<const N: u64>(&mut self) {
        self.total += N;
    }
}

let mut counter = Counter { total: 0 };
reify_add(5, &mut counter);
reify_add(10, &mut counter);
assert_eq!(counter.total, 15);
```

## V1 Limitations

**Return types that depend on N:**

```rust
#[reifiable(range = 0..=255)]
trait ArrayMaker {
    fn make<const N: u64>(&self) -> [u8; N];  // ERROR!
}
```

This won't compile — the dispatch function would need a single return
type, but `[u8; N]` is different for each `N`. For these cases, use
the manual `NatCallback` approach from Guide 3.

**Multiple const parameters per method:**

```rust
#[reifiable(range = 0..=255)]
trait BinaryOp {
    fn op<const A: u64, const B: u64>(&self) -> u64;  // ERROR in V1
}
```

Use `Nat2Callback` + `reify_nat2` for binary dispatch (Guide 3).

## How It Compares

| Approach | Boilerplate | Const generic? | Auto-dispatch? |
|---|---|---|---|
| Manual `NatCallback` impl | High | Yes | No |
| `def_nat_callback!` macro | Low | Yes (expressions) | No |
| `reify_nat_fn` closure | None | No | No |
| `#[reifiable]` on trait | None | Yes (full) | Yes |

`#[reifiable]` is the highest-level API: you define a trait normally,
add one attribute, and get dispatch functions for free.
